# 文档索引模式详解（精简模式 / 平衡模式）

> 目的：用自然语言把“上传后到底做了什么、怎么做的、涉及哪些提示词”讲清楚。
>
> 说明：本文聚焦上传 PDF 后的索引主流程，因为两种模式的差异主要在 PDF 索引环节。其他格式会在文末单独说明。

---

## 1. 从上传开始：系统先做哪些通用动作

无论你选精简还是平衡，上传后都会先走同一套基础步骤：

1. **保存文件并建档**
   - 文件落盘，写入文档元信息（名称、大小、状态、时间等）。
   - 初始状态进入待处理队列。

2. **异步启动索引任务**
   - 索引在后台线程/协程执行，不阻塞上传接口。
   - 文档状态会按阶段更新（排队 -> 索引中 -> 写索引 -> 完成/失败）。

3. **预读页数（PDF）并更新进度基线**
   - 系统会先读 PDF 总页数，作为进度显示和后续质量校验的基线。
   - 同时启动一个“估算进度线程”，让前端有可见进度反馈。

4. **应用当前模式配置**
   - 每次索引任务开始时，系统会读取当前设置页里的模式（精简/平衡）。
   - 然后把该模式对应参数装配到本次任务。

---

## 2. 精简模式：每一步具体做什么

精简模式目标是：**优先速度，保证可检索可预览**。

### Step A：按页提取文本并计算 token

- 系统逐页抽取 PDF 文本，形成“页文本 + token 数”的列表。
- 这一份数据是后面所有 TOC 判断、分段、索引写入的输入底座。

### Step B：先判断“是否存在目录页”

- 系统先做批量目录检测（一次检查多页），减少调用次数。
- 若批量检测异常，会回退到单页检测。

**涉及提示词（目录检测）**
- 目标：判断页面是否为目录页。
- 典型输出：`是否检测到目录`、`置信度`、`简要理由`。

### Step C：根据目录情况选择处理路径

#### 路径 1：检测到目录且目录里带页码

系统会执行：

1. 把原始目录文本转换成结构化目录（层级、标题、页码）。
2. 在正文开头若干页里定位目录项对应的“真实物理页”。
3. 计算目录页码和物理页码的偏移量并回填。
4. 对缺失页码的条目做补偿定位。

**涉及提示词（目录已带页码）**
- 目录结构化转换提示词（把原始目录转成 JSON 层级）。
- 目录项物理页定位提示词（把目录标题映射到正文页面标签）。
- 缺失页码补全提示词（在上下文页范围内补全未定位条目）。

#### 路径 2：检测到目录，但目录不带页码

系统会执行：

1. 先把目录文本转换为层级结构（无物理页）。
2. 分组读取正文，在每组中判断各标题是否“在本组起始出现”。
3. 把结果累积成带物理页的目录结构。

**涉及提示词（目录不带页码）**
- 目录结构化提示词（提取层级与标题）。
- 分组页码补全提示词（判断条目是否在当前分组开头出现并返回物理页）。

#### 路径 3：没检测到可用目录（或前两条失败）

系统会执行“规则优先，LLM 兜底”：

1. 先用规则抽取章节候选（中文章/节、数字层级、财报锚点等）。
2. 做噪声过滤（页眉页脚、表格行、纯数字行、重复行）。
3. 如果规则结果足够可靠，直接采用。
4. 如果规则可靠性不足，才走 LLM 生成目录（init + continue 分段续写）。

**涉及提示词（无目录路径）**
- 目录初始化生成提示词（从首组文本直接抽层级目录）。
- 目录续写提示词（在后续分组继续补齐结构）。

### Step D：目录结果质量校验与自动回退

无论走哪条路径，系统都会进行统一质量检查：

- 页码有效性校验（超出文档范围会截断/剔除）。
- 结构质量评分（覆盖度、顺序、异常项等）。
- 按评分自动回退：
  - 高分直接接受。
  - 中分尝试降复杂度路径重算。
  - 低分继续回退，直到最后兜底返回可用结果。

### Step E：标题“起始位置”复核 + 大节点递归细分

1. 对每个目录项做“是否出现在页开头”的复核。
2. 对过大的章节做递归切分，避免一个节点覆盖太多页。

**涉及提示词（复核）**
- 标题是否出现在页面中。
- 标题是否从页面开头开始。

### Step F：索引增强（精简模式版本）

精简模式只做必要增强：

- 写入节点 ID。
- 写入节点正文文本（用于检索）。
- **不生成节点摘要**。
- **不生成文档描述**。

### Step G：落盘与收尾

- 写索引文件。
- 更新文档状态为完成。
- 触发搜索索引重建（后台）。

## 3. 平衡模式：每一步具体做什么

平衡模式前半段和精简模式一致（A~E 一样），差异集中在参数和后处理增强。

### 3.1 前半段流程

- 同样会做目录检测、路径分流、质量校验、标题复核和大节点切分。
- 同样有失败回退机制，避免一次失败直接整篇失败。

### 3.2 后处理增强（平衡模式关键差异）

在写索引前，平衡模式会多两层语义增强：

1. **节点摘要生成**
   - 对每个节点调用摘要提示词，生成一句话摘要。
   - 支持并发与重试，失败时会回退到文本截断摘要，尽量不留空。

2. **文档描述生成**
   - 基于“清洗后的结构信息”生成文档级一句话描述。
   - 用于预览信息和解释性展示。

**涉及提示词（平衡模式新增）**
- 节点摘要提示词：输入“节点标题 + 节点内容”，输出短摘要。
- 文档描述提示词：输入“结构摘要”，输出一句话文档描述。

## 4. 两种模式的关键实现差异（当前默认参数）

| 维度 | 精简模式 | 平衡模式 |
|---|---|---|
| 目录检测窗口 | 更小 | 更大 |
| 大节点切分触发 | 更早触发 | 更晚触发 |
| 节点正文文本 | 开启 | 开启 |
| 节点摘要 | 关闭 | 开启 |
| 文档描述 | 关闭 | 开启 |
| 总体目标 | 先快后全 | 先全后快 |

简化理解：

- 精简模式 = 保留检索底座，减少高耗时语义增强。
- 平衡模式 = 在检索底座上补齐语义增强与解释信息。

## 5. 提示词清单（按流程分组）

以下是索引流程中实际使用/可能使用到的提示词能力（自然语言描述）：

### 5.1 目录识别与提取

1. **目录检测（单页/批量）**
   - 问题：这一页是不是目录页？
   - 输出：是/否 + 置信度。

2. **目录是否包含页码**
   - 问题：目录文本里是否已有页码信息？
   - 输出：是/否。

3. **目录结构化转换**
   - 问题：把原始目录文本转成层级 JSON（标题、层级、页码）。
   - 输出：结构化目录项。

4. **目录续写**
   - 问题：当前分组不够，继续补齐后续目录结构。
   - 输出：增量目录项。

5. **目录完整性检查**
   - 问题：当前转换结果是否完整。
   - 输出：完整/不完整（驱动重试续写）。

### 5.2 页码与锚点定位

1. **目录项物理页映射**
   - 问题：目录中的标题在正文哪一页开始？
   - 输出：带物理页标签的目录项。

2. **缺失页码补全**
   - 问题：在上下界范围内补齐没有物理页的目录项。
   - 输出：补全后的目录项。

### 5.3 标题校验与边界修正

1. **标题是否出现**
   - 问题：给定标题是否出现在给定页面。
   - 输出：是/否。

2. **标题是否页首开始**
   - 问题：给定标题是否在页面开头出现。
   - 输出：是/否。

### 5.4 语义增强（平衡模式）

1. **节点摘要生成**
   - 问题：把节点内容压缩为一句摘要。
   - 输出：短摘要。

2. **文档描述生成**
   - 问题：根据整体结构生成一句文档描述。
   - 输出：文档级描述。

## 6. 失败与回退：为什么通常不会“整篇崩掉”

系统当前有多层防护：

1. **步骤级回退**
   - 有目录+页码失败 -> 回退到“目录无页码路径”。
   - 目录无页码失败 -> 回退到“无目录路径”。

2. **质量级回退**
   - 输出质量不达标，会自动降复杂度重算。

3. **时间级保护**
   - 文档索引有全局超时上限，超时会明确失败状态，而不是无限等待。

4. **调用级保护**
   - LLM 调用有超时、重试、间隔控制，避免瞬时波动直接打穿流程。

## 7. 你关心的重点：当前可改进点（实现导向）

以下建议聚焦“提高精简/平衡两种模式的成功率与可用性”，不依赖 Auto 模式：

### 7.1 精简模式改进建议

1. **保留轻量摘要（可选）**
   - 不是全量摘要，而是只对一级节点生成极短摘要。
   - 目的：几乎不显著增时，但改善抽象问法召回。

2. **无目录路径增加硬规则权重**
   - 对财报、规范类文档提升规则锚点优先级，减少 LLM 兜底频次。

3. **目录输出结构强校验**
   - 精简模式最怕坏 JSON 带来“快但不可用”，应在落盘前做 schema 修复。

### 7.2 平衡模式改进建议

1. **摘要生成分层并发**
   - 先生成高层节点摘要，再按预算生成子节点摘要。
   - 目的：减少长尾耗时和超时重试。

2. **文档描述异步化**
   - 描述生成失败不阻塞主索引完成；后续后台补写。

3. **预览路径不做重任务**
   - 预览应该“快展示”，不要同步触发重生成任务。

### 7.3 两模式共通改进建议

1. **统一失败码体系**（格式错误、TOC不一致、页码偏移失败、超时等）。
2. **统一质量分落库**（结构质量、空节点率、校验失败次数）。
3. **统一降级链路可观测**（每次回退记录“从哪到哪、为什么”）。
4. **建立分类型回归集**（扫描件/财报/学术/手册），按周回归调参。

## 8. 非 PDF 文档说明（简要）

当前两模式的显著差异主要发生在 PDF 主链路。

- Markdown、Docx、Txt、表格等类型，主要走多格式适配器的“原生解析 + 低质量兜底”逻辑。
- 这些类型目前不完全等价地继承 fast/balanced 的全部差异开关。
- 这也是后续要统一治理的重点之一（让模式在多格式上的行为更一致）。

---

## 9. 结论

- **精简模式**：通过关闭高耗时语义增强，保留检索底座，实现“快且可用”。
- **平衡模式**：在同一主链路上增加摘要和描述，换取更稳的复杂问答表现。
- 两种模式都已具备回退与超时保护，但要冲击更高稳定性目标，需要继续加强：
  - 结构输出强校验
  - 预览与重任务解耦
  - 统一可观测与失败码
