# AI Knowledge TOC 构建测试基线

> 日期：2026-06-18  
> 输入目录：`D:\chrome_download\rag-skill-main\rag-skill-main\knowledge\AI Knowledge`  
> 用途：作为 TOC 统一状态机实施时的端到端测试输入和逐阶段验收基线。

## 1. 使用方式

这份文档不是现有实现效果说明，而是下一轮重构的预期行为清单。实施时必须按阶段验证：前一阶段达到预期后，再进入下一阶段。

每个样本都要记录实际路径、关键中间产物和最终 TOC 形态。如果实际结果偏离本基线，需要先分析原因，再决定是修实现还是修基线。

## 2. 统一阶段验收门

### 2.1 S0 Analyze

每份文档必须先得到稳定的低成本画像：页数、文本覆盖率、文本质量、内置目录数量、规则 TOC 页信号。S0 不跑全文 OCR，不调用 LLM。

验收点：

- `content_type` 只能是 `text`、`ocr`、`hybrid` 三类之一。
- `embedded_toc_signal` 与内置 outline/bookmarks 是否存在一致。
- `toc_page_rule_signal` 只表示可能有可见目录页，不直接生成 TOC。

### 2.2 S1 PreprocessToPageText

所有文档统一转换为 `PageTextMap`。

验收点：

- 文本型文档直接使用 PDF 文本层。
- 扫描/乱码型文档全文 OCR。
- 混合型文档只对坏页 OCR，再按物理页拼回。
- 后续节点内容必须从 `PageTextMap` 填充，不重复 OCR。

### 2.3 S2-S4 TOC 构建

按确定性路径执行：`embedded_toc`、`visible_toc_with_pages`、`visible_toc_no_pages`、`content_outline`。

验收点：

- 不允许同时跑多条完整 TOC 路径竞赛。
- 标准目录规则抽取必须高置信；失败、低置信或校验不过时降级 LLM。
- 对没有可用 `code_toc` 但可见目录质量很高的文档，S4 应优先用规则抽取；规则结果通过完整性、页码语义和标题命中校验后，不应再调用 LLM 抽取。
- `embedded_toc` 必须同时检查 bookmarks/outline 和 PDF link annotations，不能因为 bookmarks 存在就跳过 links。
- 如果 PDF links 暴露出图目录、表目录，最终结果必须拆成独立 `toc_sections`；缺失这些辅助目录时不能提前接受主目录。
- 弱 slide-export outline 必须先清洗并通过硬校验，不能被 fast path 直接早返回；fast path、provider、质量门的判定标准必须一致。
- 图目录、表目录必须与主目录分开。
- 无页码目录不能猜 offset，必须靠正文标题定位。
- 只有顶级 anchor 时，章节内部结构交给 LLM 判断，每页传前 200 字和物理页码。

### 2.4 S5-S6 映射与质量门

最终 TOC 必须使用物理页码。

验收点：

- 页码映射支持边界重合：若 `next.start_page == current.end_page`，当前节点结束页保留为 `next.start_page`。
- 范围修正规则对所有同级节点递归适用。
- 大量节点落在目录页、页码越界、标题定位率过低必须失败。
- 原文目录平铺时，结果平铺不能被质量门判失败。

## 4. 逐文档测试基线

### 4.1 2025全球人工智能技术应用洞察报告.pdf

基础信号：43 页，文本层可用但单页文本较短，内置 outline 49 条，无明显可见目录页，形态接近幻灯片报告。实际 PDF 书签包含 `默认节`、`幻灯片 X` 等弱 slide-export 噪声，不能把 raw outline 直接视为高质量目录。

预期路径：`text` -> `embedded_toc`。

预期结果：

- 使用内置 outline 作为主目录来源。
- 过滤 `默认节`、无意义空标题等噪声。
- 保留章节分组和幻灯片页级节点。
- 关键章节起始页应包括：`Chapter 1` 物理页 4、`Chapter 2` 物理页 11、`Chapter 3` 物理页 24、`Chapter 4` 物理页 38。
- fast path、provider、质量门对弱 slide-export outline 的判断必须一致：清洗后通过硬校验才可接受；如果只拿到 raw 幻灯片书签，应拒绝早返回。
- 不应进入全文 OCR，也不应调用 LLM 跑完整目录抽取。

阶段验收：S2 必须记录 raw outline 数量、清洗后数量和拒绝/接受原因；S5 页码全部在 1-43 范围内；S6 不因合理的幻灯片平铺而失败，但必须拒绝 `默认节` 等噪声占主导的结果。

### 4.2 2025年AI治理报告：回归现实主义.pdf

基础信号：76 页，PDF 文本层覆盖率较高但正文和目录页明显乱码，内置 outline 11 条且相对可用。

预期结果：

- S2 使用内置 outline 生成主目录。
- 最终节点内容必须来自 OCR 后文本，而不是乱码 PDF 文本层。
- 顶级结构应包括 `2025 AI治理年度洞察`、`2025国际AI治理总体态势`、`AI治理主要议题：回顾与展望` 等主要部分。

阶段验收：S1 乱码页不能直接进入后续正文填充；S2 outline 页码必须通过 OCR 文本抽样校验；S6 不能接受节点内容大面积乱码的结果。

### 4.3 2025年度重庆市人工智能应用场景典型案例集（压缩版）.pdf

基础信号：44 页，文本覆盖率约 0，扫描型 PDF，存在视觉目录页，目录页包含案例编号、标题和页码。

预期路径：`ocr` -> `visible_toc_with_pages`。

预期结果：

- S1 全文 OCR，按物理页生成 `PageTextMap`。
- S3 在 OCR 文本上识别目录页。
- S4 从目录页抽取案例目录；规则高置信时可先尝试规则抽取，失败则 LLM 抽取。
- `layout_required` 只允许触发 S1 OCR；S4 必须继续使用 OCR 后的 `PageTextMap` 走 `visible_toc_with_pages`，不能跳到旧 layout/VLM 目录路径。
- 若规则抽取能得到目录原始项但映射校验失败，必须降级到 LLM 目录页文本抽取；不能直接返回 0 项或单节点。
- 目录应保留案例编号和案例标题，不能只剩一个覆盖全文的大节点。
- 页码映射必须基于内容标题校验；不能大量节点落在目录页。

阶段验收：S1 OCR 覆盖 44 页；S3 命中目录页约为物理页 2；S4 抽出的案例条目数量应接近原目录规模；S5 标题定位率必须达标；S6 大量节点指向目录页时失败。

预期结果：

- 虽然内置 outline 可用，但如果只接受 bookmarks 会缺失图目录；S2 必须继续检查可见目录页 links 或其他代码信号。只有 embedded TOC 补齐图目录后才能通过，否则必须进入可见目录路径。
- 进入可见目录路径后，S4 应优先规则抽取主目录和图目录；规则拆分失败或校验不通过时才回退 LLM。
- 最终前端顶级节点应至少包括：`目录`、`图目录`。
- 主目录应包含人工智能安全治理概述、全球治理现状、核心挑战、产业实践、发展建议等主要部分。
- 图目录作为辅助索引独立映射。

阶段验收：S2 不能只因内置 outline 可用就提前接受缺失图目录的结果；如果 S2 接受，则必须记录图目录来源；若进入 S4，可见目录规则抽取通过时不调用 LLM；S5 主目录和图目录独立映射；S6 缺失图目录时失败或至少标记不通过。

### 4.12 清华大学：职业教育人工智能应用发展报告（2024-2025）.pdf

基础信号：201 页，文本层可靠，无内置 outline，物理页 2 是无页码目录。

预期路径：`text` -> `visible_toc_no_pages`。

预期结果：

- S3 应识别物理页 2 为目录页。
- S4 抽取 `序言` 和 8 个章节标题。
- 不猜页码，不计算 offset，通过标题检索定位章节起始页。
- 每个章节内部由 LLM 根据该章节范围内每页前 200 字生成子目录结构。
- 最终不应只有 9 个顶级节点；每章下应有可用子结构或页级子节点。

阶段验收：S5 顶级标题定位率必须达标；S6 如果多个顶级章节范围覆盖到文档末尾且互相大范围重叠，应失败。

### 4.13 生成式人工智能服务合规备案指南（2026年）.pdf

基础信号：70 页，文本层可靠，内置 outline 172 条，目录页为物理页 7-9，其中物理页 7 是主目录，物理页 8 是表目录，物理页 9 是图目录。实际 PDF 同时包含 bookmarks 和目录页 link annotations：bookmarks 覆盖主目录，links 能暴露主目录、表目录、图目录。

预期路径：`text` -> `embedded_toc`。

预期结果：

- S2 必须独立采集 bookmarks 和 PDF links，不能因为 bookmarks 存在就跳过 links。
- S2 应把 bookmarks 与 links 合并成 `toc_sections`，区分主目录、表目录、图目录。
- 如果 embedded TOC 不能生成表目录/图目录顶级节点，或生成的主目录结构/长叶子质检失败，就不能直接作为最终结果，必须降级到可见目录路径。
- 最终前端顶级节点应为：`目录`、`表目录`、`图目录`。
- 主目录应保留多级结构：章、节、括号编号条目。
- links 目标页可直接作为物理页，但仍需抽样标题锚点校验；已知 `第一章总则` 应对应物理页 10。
- 表目录和图目录独立映射，不能全部落在目录页 8 或 9。

阶段验收：S2 必须记录 bookmarks 条数、links 条数、link 页范围 7-9，以及三类 `toc_sections`；S5 `第一章总则` 应映射到物理页 10；表目录/图目录项必须通过标题或编号在正文中定位；S6 缺失图目录或表目录时失败。

## 7. 通过标准

一份样本只有同时满足以下条件才算通过：

- 实际路径符合本基线，或偏离路径有明确且合理的解释。
- 每个阶段的中间产物可检查，不是直接只看最终 JSON。
- 最终 TOC 使用物理页码。
- 节点内容来自 `PageTextMap`。
- 主目录、图目录、表目录按需分开。
- 页码映射和节点范围通过抽样校验。
- 质量门没有把明显错误结果放行。
