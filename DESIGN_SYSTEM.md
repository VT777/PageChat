# PageChat Design System

日期：2026-06-23  
状态：v4，根据文档预览、设置项精简和聊天主页面预览反馈修订

## 1. 设计方向

PageChat 的前端应采用 **PageIndex-like product structure + Apple-level polish**。

这里的 Apple 风格只指材料、精密度和克制感，不代表复制 macOS 窗口或 Apple 官网视觉。产品结构优先参考官方 PageIndex：左侧全局导航栏、主工作区、简洁文件列表、轻量弹窗、清晰菜单和紧凑操作。

设计目标：

- 统一登录页、聊天页、文档管理页、设置页的视觉语言。
- 主聊天页优先参考官方 PageIndex 的对话流：工具调用是回答内容中的内联步骤，不是独立卡片。
- 优先重做设置页和文档管理页。
- 保留工具型产品的密度，不做营销式页面。
- 使用浅灰背景、白色工作面、细边框、轻阴影和蓝色动作色。
- 不照抄 macOS 窗口细节，不出现红黄绿三色窗口控制点。

## 2. 产品气质

界面应该给用户这些感受：

- **清爽：** 页面留白充足，但不空洞。
- **可信：** 文档状态、解析路径、耗时、工具调用证据都能被检查。
- **高效：** 上传、搜索、选择、批量操作、预览都直接可用。
- **统一：** 左侧导航、顶栏、列表、弹窗、输入框属于同一套系统。
- **克制：** 不用装饰性渐变、光球、大卡片来制造“高级感”。

避免：

- 复制 macOS 窗口三色按钮。
- 在聊天回答里用独立大方框展示工具调用。
- 使用右侧固定信息栏作为文档管理页主结构。
- 把“批量模式”做成单独切换项。
- 在设置页把 Model Providers 做成任务模型路由。
- 设置页弹窗过窄，导致供应商列表和配置表单拥挤。
- 使用紫色/蓝紫渐变作为主视觉语言。
- 卡片套卡片。
- 大圆角泛滥。

## 3. 全局布局

### 3.1 App Shell

登录后的页面统一使用官方式左侧全局导航：

- 顶部：产品图标。
- 主导航：New Chat、Documents。
- 中段：Chats 列表。
- 底部：Settings、辅助入口。
- 右侧：当前页面主工作区。

左侧导航栏是全局结构，不应在文档管理页里再做一套不同的文件夹侧栏作为主导航。文档文件夹层级应放在主区域面包屑、列表、创建文件夹和移动弹窗中。

桌面尺寸建议：

| 元素 | 规则 |
| --- | --- |
| 全局导航栏 | `280px` 左右，接近官方截图 |
| 页面顶栏 | `52px` 高，包含页面标题和账号区 |
| 主内容最大宽度 | 文档列表区域可居中，宽度约 `960px` 到 `1040px` |
| 页面背景 | `#f6f7f9` 或接近浅灰 |
| 主面板 | 白色或近白，细边框 |

### 3.2 顶栏

顶栏用于：

- 页面标题。
- 账号入口。

顶栏不应显示 `Upgrade`，除非产品真的上线计费能力。当前阶段取消该入口，避免界面像未完成的 SaaS 模板。

顶栏不使用 macOS 三色点。任何窗口或弹窗也不使用三色点。

## 5. 文档管理页

文档管理页应优先参考官方截图。

### 5.1 页面结构

结构：

- 左侧：全局导航栏，包含 New Chat、Documents、Chats 列表、Settings。
- 右侧主区域：
  - 顶栏：与 Chat 页面一致显示当前页面标题，文档管理页显示 `Documents`，但不显示 `Share`。
  - 工具栏：Upload、Create Folder、Search、视图切换。
  - 面包屑：使用 `root > 当前文件夹` 格式，根路径显示为 `root`。
  - All Files 行，带默认 checkbox。
  - 文档/文件夹列表。

文档管理页是主功能页，不应像嵌入在页面中的小卡片：

- 主区域内容应占满可用宽度，保持合理左右页边距，而不是窄宽居中面板。
- 文件列表使用页面级分隔线和轻背景，不使用厚边框大卡片包裹。
- 列表顶部 `All Files` 行是列表的一部分，边界应轻，不制造明显“内嵌框”。
- 页面内部不再重复显示 `Documents` 大标题，第一行显示当前路径 breadcrumb。
- 文件夹行使用文件夹图标和 item count，点击文件夹进入下一级，breadcrumb 同步更新。
- 文件夹内容超出可视区域时，文件/文件夹列表区域必须支持独立滚动。

取消：

- 现有右侧固定信息栏。
- “批量模式”切换按钮。
- 页面顶部堆叠过多统计卡片。
- 顶栏中的 `Share` 入口。

### 5.2 选择和批量操作

批量选择是默认能力，不是模式。

规则：

- 每一行左侧都有 checkbox。
- All Files 行有总 checkbox。
- 用户选中一个或多个文档后，工具栏出现批量操作区。
- 批量操作包括 Move、Delete、Reprocess、Download，可按权限和状态启用。
- 未选中时不显示大块批量 UI。

### 5.3 文档行

每个文档行包含：

- checkbox。
- 缩略图或文件类型图标。
- 文件名。
- 页数或文件类型。
- 上传时间。
- 行内快捷动作：Chat、More。

交互：

- 点击文件名或行主体打开文档预览弹窗。
- checkbox 点击只改变选择状态，不打开预览。
- More 打开菜单，包含 Chat、Move To、Copy ID、Download、Share、Reprocess、Delete。
- hover 只显示轻背景和行内操作，不改变行高。

### 5.4 文档预览弹窗

点击文档后打开弹窗，类似当前预览，但信息更完整。

布局：

- 弹窗居中，占据大部分桌面视口。
- 顶部：文档名、状态、关闭按钮。
- 左侧：窄 tab rail + 内容面板。
- 左侧 tab rail 至少包含两个按钮：TOC、信息。
- 默认打开 TOC，显示目录树。
- TOC 界面只显示目录，不放 `Info tab preview` 或任何元信息预览块。
- 点击信息后，左侧内容切换为文档元信息和处理信息。
- 右侧：只显示文档预览，不显示元信息卡片、解析信息卡片或其他 inspector。

必须补充的元信息：

- 文件名、原始文件名、文件类型、文件大小。
- 页数。
- 上传时间、更新时间。
- 当前状态。
- 索引/解析路径，例如 `pdf_text_layer -> llm_toc_validation`、`ocr -> visual_summary -> toc_mapping`。
- 解析总用时。
- OCR 用时、TOC 用时、视觉摘要用时，如后端有数据。
- TOC 节点数。
- 文本字符数。
- 摘要覆盖率。
- 质量报告或质量等级。
- 处理步骤记录。

预览弹窗不是右侧信息栏的替代皮肤，而是文档详情的主入口。所有详情信息都在左侧信息 tab 中承载，右侧保持纯文档阅读和页面预览。

## 6. 设置页

设置页采用官方式弹窗/窗口布局，但内容必须服务 PageChat 的真实系统行为。

设置页不是窄弹窗。它应像 Dify 设置页一样覆盖主页面的大部分区域：

- 遮罩覆盖全屏。
- 设置窗口宽度建议 `min(1280px, calc(100vw - 96px))`。
- 设置窗口高度建议 `calc(100vh - 80px)`。
- 左侧设置导航约 `260px`。
- 右侧内容区应能容纳供应商卡片、模型列表和配置表单。
- 右上角使用普通关闭按钮，并显示 `ESC` 提示。
- 背景页面可以轻微变暗，但不要过暗。
- 内容不能挤成单列窄表单。

### 6.1 分类

推荐分类：

- 模型供应商（模型统一入口）
- OCR 设置
- 解析设置
- 问答设置
- 语言
- Account

不做：

- Model Routing 作为独立左侧入口。
- Members。
- Data Sources。
- API Extensions。
- Billing。
- Usage。
- Privacy deletion。
- Experimental features。

模型路由能力不消失，但不作为左侧一级设置项。默认聊天模型、OCR 模型、解析模型、问答模型等选择，应分别放在 OCR 设置、解析设置、问答设置中；供应商、凭据、endpoint 和可用模型统一在模型供应商中管理。

### 6.2 Model Providers

Model Providers 参考 Dify 的供应商配置方式：它是供应商、凭据、模型能力和 OpenAI-compatible endpoint 管理，不是具体任务模型设置页。

供应商列表应包含：

- OpenAI。
- Anthropic。
- Google Gemini。
- Azure OpenAI。
- Ollama 或本地模型。
- OpenAI Compatible。

每个供应商卡片展示：

- 供应商名称。
- 状态：Configured / Not configured / Error。
- 支持能力：LLM、Vision、Embedding、Rerank、OCR、Speech 等。
- 已配置模型数量。
- Configure / Edit / Test 按钮。
- 可展开的模型列表。
- 模型行应展示模型名、能力 badge、上下文长度、启用开关。

供应商页面视觉应接近 Dify：

- 顶部为标题 `模型供应商` 和右侧搜索框。
- 已配置供应商按纵向列表展示，不使用双列卡片堆叠。
- 每个供应商行包含图标、名称、版本 badge、更多菜单、能力 badge、右侧 API Key/额度状态和配置按钮。
- 已展开供应商在下方显示模型列表，模型行包含模型名、能力 badge、上下文长度和启用开关。
- 未配置供应商放在 `待配置` 分组，底部提示配置 API 密钥和添加模型。

OpenAI Compatible 配置应支持：

- Provider display name。
- API key。
- Base URL。
- Default headers，如需要。
- 模型列表，可以手动添加或从接口拉取。
- 每个模型的能力标记：chat、vision、embedding、rerank、tool calling、structured output。
- Test connection。

API key 配置弹窗参考 Dify：

- 标题：API 密钥授权配置。
- 字段：凭据名称、API Key、自定义 API endpoint 地址。
- 底部显示加密/存储说明。
- 主动作是保存，次动作是取消。
- 弹窗宽度不小于 `760px`，避免表单拥挤。
- 弹窗底部可以使用浅灰固定说明栏，展示 `PKCS1_OAEP` 加密存储提示。

参考依据：

- [Dify Model Providers 文档](https://docs.dify.ai/en/guides/model-configuration/interfaces)：workspace 级模型访问配置，流程是 Settings -> Model Providers、选择供应商、添加凭据、测试并保存。
- Dify 支持系统供应商和自定义供应商；自定义供应商使用用户自己的 API key。
- Dify 官方文档说明 provider 可配置 API key，OpenAI 配置可包含 custom base URL；这支持 PageChat 的 OpenAI-compatible provider 设计。

### 6.3 模型选择归属

任务模型选择从 Model Providers 中分离，但不作为独立左侧入口。

归属规则：

- OCR 模型放在 OCR 设置。
- TOC / 结构解析模型放在解析设置。
- 默认聊天、问答生成、引用策略、工具策略放在问答设置。
- Embedding、Rerank 等检索模型可以放在问答设置的高级分组，或在后续独立为检索设置，但当前不新增左侧入口。
- 所有任务模型选择只能选择已在模型供应商中可用且能力匹配的模型。

## 7. 聊天页

聊天主页面要参考主流 AI 对话界面，但保持 PageChat 的文档工作流特性。

### 7.1 主页面结构

主聊天页参考官方 PageIndex：

- 左侧使用全局导航栏。
- 主内容区居中，回答流最大宽度约 `860px` 到 `920px`。
- 顶栏只保留必要操作，例如 Share、账号入口；取消 `Upgrade`。
- 用户消息右对齐，使用蓝色气泡。
- 助手回答左对齐，以自然段和结构化内容为主。
- 主内容区必须支持滚轮滚动，长工具参数或长回答不能被截断。
- 输入框固定在底部 composer 区，带轻阴影和圆角；它不属于消息滚动流，不能被展开的工具参数挤出视口。
- 消息流底部需要保留足够空间，避免最后一条回答被 composer 遮挡。
- Chat shell 的父级 grid/flex 容器必须设置 `min-height: 0` 和必要的 `overflow: hidden`，否则长工具参数会撑高父容器，把 composer 挤出视口。
- 只有消息流容器负责 `overflow-y: auto`；不要让整个页面或主工作区滚动。
- 滚动容器应占满主工作区宽度，滚动条贴近主工作区最右侧；居中的回答列只负责内容宽度，不承载滚动条。
- 滚动条使用薄样式，轨道透明，避免侵占阅读区域。

### 7.2 内联工具调用

工具调用不能是独立的大方框。它应作为回答流中的内联动态步骤出现，类似官方截图：

- Thought 行：`Thought for 2 seconds`，带小图标和展开箭头。
- 工具行：`Viewed folder structure · 1 folder`。
- 工具行：`Browsed documents · 3 documents`。
- 工具行：`Read the document structure of "...pdf"`。
- 每一行都是轻量 inline row，不使用卡片背景。
- 每个工具调用必须独占一行，不允许多个工具调用横向并排。
- 展开箭头紧跟在工具动作、摘要或目标文本后面，不统一吸附到整行最右侧。
- 工具行默认只显示动作、目标和摘要。
- 点击箭头后，在该行下方展开紧凑详情。
- 展开详情参考官方工具面板：在工具行下方缩进显示 `PARAMETERS` 和 `RESULT` 两段。
- `PARAMETERS` / `RESULT` 标签使用小号大写弱色文字。
- 参数和结果使用浅灰 JSON inspector 面板展示，面板无重边框、无卡片阴影、背景接近页面底色。
- JSON inspector 可以折叠长数组或对象，例如 `structure: [ 0: {...}, 1: {...} ]`，避免展开全文、base64 或大体积结果。
- 展开详情要和官方工具行视觉统一：保持缩进、弱色文字、低边界感，不做总结 chip 或输入框式容器。
- 工具调用状态可以有 running、success、error 三种视觉状态。
- 工具调用图标使用统一线性图标，不使用 emoji、Unicode 装饰符号或临时字符。
- 推荐映射：Thought 使用 `Sparkles`，folder structure 使用 `FolderTree`，browse documents 使用 `Files`，document structure 使用 `ListTree`，read pages 使用 `BookOpen` 或 `FileText`，view figure 使用 `Image`。

### 7.3 输入框能力

对话框必须支持：

- 上传截图。
- Web search 开关。
- 选择文件。
- 选择文件夹。
- 显示当前检索范围。
- 清除已选范围。
- 发送。

推荐结构：

- 输入区域：多行输入，支持粘贴截图。
- `+` 是唯一功能入口。
- 点击 `+` 后展开菜单，菜单包含添加图片、网页搜索、选择文件、选择文件夹。
- 菜单锚定在 `+` 按钮上方，宽度和行高要保证中文菜单项不被图标、鼠标指针或边界挤压。
- 不在输入框底部平铺 Attach、Web、Files、Folders 等多个等权按钮。
- 右侧只有发送按钮，发送按钮固定在输入框右下角，不能被 chip 数量推挤位置。
- 不提供语音输入入口。
- 已选功能和上下文以内嵌形式显示在输入框中：图片缩略图、Web search chip、文件 chip、文件夹 chip。
- 已选项都必须可以取消，但 chip 默认显示功能语义图标；hover chip 后原图标变成 `X`，点击该图标才取消。
- 输入框默认保持紧凑高度；只有添加图片/截图缩略图后才展开附件区域。

推荐功能菜单：

- 点击 `+` 展开小菜单：
  - 添加图片：打开系统文件选择器，限制为图片类型；也支持粘贴截图。
  - 网页搜索：toggle，选中后在输入框内显示 `× 搜索` chip。
  - 选择文件：打开文档 picker。
  - 选择文件夹：打开文件夹 picker。

选中后的输入框效果：

- 添加图片后，在输入框上方区域显示图片缩略图，右上角有删除 `x`。
- 图片缩略图的删除按钮必须在缩略图边界内，不允许外溢。
- 开启网页搜索后，在输入框底部显示可取消的搜索 chip，样式类似 `× 搜索`。
- 选择文件后显示文件 chip，例如 `× 2 files` 或具体文件名。
- 选择文件夹后显示文件夹 chip，例如 `× Folder: test`。

选择文件/文件夹交互：

- 点击范围 chip 或文件夹按钮打开 picker。
- picker 支持搜索、最近文档、文件夹树。
- 选中后在输入框上方或下方显示 scope chip。
- scope chip 文案要短，例如 `Folder: test`、`3 files`。

上传截图交互：

- 支持按钮选择和剪贴板粘贴。
- 截图以小缩略图显示在输入框上方。
- 可以删除。
- 发送时作为视觉上下文传给模型。

Web search 交互：

- 使用 toggle chip。
- 开启后 chip 高亮。
- 最终消息中记录该次对话启用了 Web search。

### 7.4 工具调用结果

工具调用块应展示为用户可理解的步骤：

- Viewed folder structure。
- Browsed documents。
- Read document structure。
- Read pages。
- Viewed figure。

默认显示摘要，不直接展示完整 raw JSON、全文或 base64。视觉上必须内联，不使用独立方框。

### 7.5 左侧底部操作

- 左下角只保留设置按钮。
- 不在左下角放列表、显示器、布局切换等低频入口。
- 设置按钮使用 `SlidersHorizontal` 或同等“配置控制”语义的线性图标，不使用容易被误认为明暗主题的太阳/放射状图标。

## 8. 登录页

登录页保持产品一致性：

- 左侧可以放产品预览，而不是抽象渐变。
- 右侧登录/注册表单。
- 不使用紫色渐变背景。
- 不使用装饰光球。
- 不复制 macOS 窗口控件。

## 9. 组件规则

### 9.1 按钮

常见命令优先使用 lucide 图标：

- Upload。
- Search。
- Settings。
- Delete。
- Move。
- Refresh。
- Preview。
- Close。
- Back。
- Web search。
- Attach。

图标规范：

- 正式实现统一使用 `lucide-vue-next`。
- 图标默认尺寸 `18px`，紧凑场景 `16px`，主动作按钮 `20px`。
- 线宽保持一致，建议 `1.75px` 到 `2px`；不要混用实心、emoji、Unicode 符号、品牌字母块和临时 SVG。
- 同一行内图标必须占固定容器宽度，建议工具调用、菜单、侧栏行都使用 `24px` 到 `28px` 的 icon cell，避免文本起点错位。
- 图标视觉复杂度要一致：优先选简单线性轮廓，避免过度复杂的齿轮、层叠文件、装饰星形等图标抢占注意力。
- 不使用 emoji、Unicode 方块、箭头、星号等字符充当功能图标。
- chip 中的取消使用 `X` 图标，发送使用 `ArrowUp`，添加入口使用 `Plus`。
- chip 内不要把取消图标和语义图标挤在一起：取消按钮放在独立圆形 `18px` 区域，语义图标跟随其后，文本从统一位置开始。

推荐映射：

- 添加入口：`Plus`。
- 发送：`ArrowUp`。
- 关闭/取消：`X`。
- 上传截图/图片：`ImagePlus`。
- 网页搜索：`Globe`。
- 选择文件：`FileText` 或 `File`。
- 选择文件夹：`Folder`。
- 新对话：`MessageSquare`。
- 文档管理：`FileText`。
- 设置：`SlidersHorizontal` 或 `Settings2`，优先用滑杆类图标；不要使用太阳、放射线或容易被误认为主题切换的图标。
- Thought：`SunMedium`、`Sparkle` 或项目自定义的简化 reasoning 图标，但必须与其他线性图标同线宽。
- 查看文件夹结构：`FolderTree`。
- 浏览文档：`FileSearch`。
- 读取文档结构/TOC：`ListTree`。
- 查看页面/正文：`BookOpen` 或 `FileText`。
- 查看图片：`Image`。

按钮高度：

- 紧凑：`28px`。
- 默认：`32px`。
- 输入框主动作：`36px` 到 `40px`。

### 9.2 菜单

菜单参考官方截图：

- 白色背景。
- 细边框。
- 轻阴影。
- 图标 + 文本。
- destructive 动作使用红色。
- 分组之间使用分隔线。

### 9.3 列表

列表比卡片更适合文档管理。

规则：

- 行高稳定。
- hover 显示浅灰背景。
- 选中状态不改变布局。
- 文件缩略图尺寸稳定。
- 长文件名单行省略。

### 9.4 弹窗

弹窗不使用三色窗口按钮。

规则：

- 顶部使用标题、辅助信息、关闭按钮。
- 关闭按钮是普通 `X` 图标按钮。
- 弹窗背景用白色或浅材料面。
- 遮罩轻量，不要过暗。

## 10. 迁移计划

### Phase 1：设计系统底座

- 添加 CSS 变量和通用 utility class。
- 建立共享 app shell。
- 建立按钮、输入框、菜单、列表行、弹窗、segmented control 样式。
- 更新设计预览页。

### Phase 2：文档管理页

- 采用官方式文档列表结构。
- 取消右侧固定信息栏。
- 移除批量模式切换，改为默认 checkbox 选择。
- 行点击打开文档预览弹窗。
- 预览弹窗左 TOC、右文档预览，并补充解析路径、耗时、质量等元信息。

### Phase 3：设置页

- 设置页改为设置窗口布局。
- 设置窗口覆盖主页面大部分区域，避免窄挤。
- Model Providers 改为 Dify-like 供应商配置。
- 左侧设置项精简为模型供应商、OCR 设置、解析设置、问答设置、语言、Account。
- OCR、解析、问答页面分别承载各自任务模型选择和真实可生效配置。

### Phase 4：聊天页

- 更新主输入框。
- 工具调用改为内联动态步骤，可展开查看详情。
- 支持截图上传、Web search、文件/文件夹选择。
- 所有增强能力收进 `+` 菜单，输入框底部不平铺多个功能按钮。
- 工具调用块改为官方式工作流摘要。

### Phase 5：登录页

- 收敛到同一视觉系统。
- 用产品预览替代装饰渐变。

## 11. 验收清单

- 文档管理页左侧使用全局导航栏。
- 文档管理页无右侧固定信息栏。
- 批量选择默认可用，没有“批量模式”入口。
- 文档行点击打开预览弹窗。
- 预览弹窗左侧有 TOC / 信息两个按钮。
- 预览弹窗默认显示 TOC，点击信息后左侧切换为元信息。
- 预览弹窗右侧只显示文档预览。
- 文档元信息包含解析路径和解析用时。
- 设置页 Model Providers 是供应商配置，不是任务模型路由。
- 设置页左侧不出现 Model Routing、成员、数据来源、API 扩展。
- 设置页覆盖主页面大部分区域，不是窄弹窗。
- API key 配置弹窗参考 Dify，包含凭据名、API Key、自定义 endpoint。
- OpenAI Compatible 供应商可配置 base URL、API key 和模型能力。
- 任务模型选择按用途放到 OCR 设置、解析设置、问答设置。
- 所有窗口和预览都不出现 macOS 三色点。
- 聊天主页面不显示 Upgrade。
- 聊天工具调用以内联步骤展示，可展开，不使用独立大方框。
- 聊天输入框支持截图、Web search、文件选择、文件夹选择。
- 聊天输入框只保留 `+` 作为功能入口。
- 聊天输入框不出现语音输入。
- 已选搜索、文件、文件夹和图片都以内嵌 chip 或缩略图显示，并可取消。
- 页面不使用装饰渐变光球。
- 卡片圆角不超过 `8px`，弹窗外壳不超过 `10px`。
- 桌面浏览器截图检查无文本溢出和元素重叠。
