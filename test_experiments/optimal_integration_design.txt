Optimal Multimodal Integration Design
================================================================================

Based on analysis, here's the optimal integration strategy:

TIER 1: Official Pipeline (Primary)
----------------------------------------
Keep official's 3-mode cascade as the main pipeline:
  Mode 1: TOC + page numbers -> use directly
  Mode 2: TOC without numbers -> text search for locations
  Mode 3: No TOC -> LLM generate structure from text

Why keep it: Simple, proven 98.7% accuracy, low cost

TIER 2: Multimodal Enhancement (Fallback)
----------------------------------------
Add VLM as a fallback when text pipeline struggles:

Trigger 1: Mode 3 + Document has divider pages
  Condition: No TOC AND _detect_chapter_dividers() finds dividers
  Action: Use VLM to analyze divider pages and generate structure
  Benefit: Handles 'hui bao ti gang' and similar patterns
  Cost: 1-2 VLM calls (thumbnail grids)
  Impact: High (saves failed Mode 3 processing)

Trigger 2: Mode 2 + TOC page numbers look wrong
  Condition: TOC extracted but page numbers are suspicious
  Action: Use VLM to verify TOC page images
  Benefit: Catches regex errors (like p.1945 for ENIAC)
  Cost: 1 VLM call per TOC page
  Impact: Medium (prevents wrong offset calculation)

Trigger 3: Verification fails
  Condition: Official verification accuracy < 60%
  Action: Use VLM for visual verification instead of text
  Benefit: More reliable than text matching
  Cost: 1 VLM call per sample
  Impact: Medium (improves accuracy for borderline cases)

Trigger 4: Text quality is low
  Condition: _check_text_quality() flags low quality
  Action: Use OCR/VLM to re-extract text before processing
  Benefit: Handles garbled/scanned PDFs
  Cost: 1 OCR call per page (expensive)
  Impact: High (saves completely failed documents)
  Note: Use sparingly, only for truly bad text

TIER 3: What NOT to do
----------------------------------------
Avoid these anti-patterns:
  X VLM as primary TOC detector (unreliable, expensive)
  X Parallel text + visual paths (wasteful, complex)
  X VLM for every document (unnecessary cost)
  X Complex branch logic (hard to maintain)
