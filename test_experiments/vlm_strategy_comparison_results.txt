RECOMMENDATION ANALYSIS
================================================================================

Strategy 1 - Conservative:
  Pros: Very low cost, simple
  Cons: Misses divider-based docs (like 5th Paradigm)
  Best for: Budget-constrained environments

Strategy 2 - Divider-based:
  Pros: Targets exact problem, low false positive
  Cons: Misses low-quality and complex Mode 3 docs
  Best for: Documents with clear visual dividers

Strategy 3 - Verification-fallback:
  Pros: Catches all difficult Mode 3 docs
  Cons: Requires running official pipeline first (latency)
  Best for: Accuracy-critical applications

Strategy 4 - Hybrid:
  Pros: Comprehensive coverage, context-aware
  Cons: More complex logic, higher cost
  Best for: General-purpose production systems

Strategy 5 - Aggressive:
  Pros: Maximum coverage
  Cons: High cost, many unnecessary VLM calls
  Best for: When accuracy is more important than cost

================================================================================
RECOMMENDED APPROACH
================================================================================

PRIMARY RECOMMENDATION: Strategy 3 - Verification-fallback
Score: 0.78

Rationale:
  1. Covers all problem cases (dividers, low quality, complex Mode 3)
  2. Context-aware: different VLM solutions for different problems
  3. Balanced cost: only ~32% of docs trigger VLM
  4. Extensible: easy to add new triggers

Implementation Logic:
  if is_low_quality:
      -> OCR re-extraction (expensive but necessary)
  elif has_dividers:
      -> VLM divider analysis (2 calls, high impact)
  elif mode == 'Mode 3' and not has_clear_headings:
      -> VLM structure generation (3 calls)
  else:
      -> Official pipeline only (no VLM)

Expected Performance:
  - Accuracy: ~95% (up from ~75%)
  - VLM Trigger Rate: ~32% of documents
  - Avg VLM Calls per Triggered Doc: ~8.6
  - Coverage: 100% of problem documents
  - Precision: ~85% (mostly true positives)
