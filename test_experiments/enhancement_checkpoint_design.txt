Unified Enhancement Interface Design
================================================================================

Core idea: Decorator Pattern
  - Keep official API unchanged
  - Insert optional enhancement checks at each step
  - Fall back to official implementation if enhancement fails

Key advantages:
  1. Official pipeline remains the primary path
  2. Enhancements are additive, not replacement
  3. Each checkpoint can be enabled/disabled independently
  4. Easy to test and measure impact of each enhancement
  5. Backward compatible - can disable all enhancements

Implementation approach:
  1. Create EnhancementProvider interface
  2. Official pipeline accepts optional EnhancementProvider
  3. Each checkpoint calls provider if available
  4. Provider returns enhanced result or None (use official)

Example:
  # Without enhancement
  result = official_pipeline.process(doc)

  # With enhancement
  provider = MultimodalEnhancementProvider()
  result = official_pipeline.process(doc, enhancer=provider)
